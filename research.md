# FireRed Romhack Research Artifact

This document consolidates research and findings for the `pokefirered` romhack project.

## 1. Move Category Icons Implementation
Research was conducted on implementing Physical/Special/Status icons in the move menu, similar to modern Pokémon games.

### Key Findings:
- **Reference Material**: Binary hacks like "FireRed 802" or "Leon's ROM Base" provide established patterns for move split icons.
- **Graphic Assets**: Icons need to be in 4bpp tiled format. Common offsets in binary hacks were identified for palette and pixel data.
- **Code Injection**: The `list_menu.c` and `summary_screen.c` files are primary targets for UI modifications.
- **Logic**: Custom logic is required to determine the category (Physical, Special, or Status) based on the move ID or a new attribute in the `gMoves` table.

## 2. ROM Expansion (32MB)
To accommodate expanded content (new Pokémon, moves, graphics), the ROM needs to be expanded from the default 16MB to 32MB.

### Implementation Steps:
- **Linker Script**: Modify `ld_script.ld` to increase the ROM size limit.
- **ROM Header**: Ensure the ROM header reflects the new size if necessary for emulator compatibility (though most GBA emulators handle 32MB automatically if the data is present).
- **Cartridge Type**: The GBA cartridge type supports up to 32MB natively.

## 3. Starter Pokémon Modification
